This skill guides creation of distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics.

## Design thinking checklist
- Clarify purpose and target audience before coding.
- Choose a bold visual direction and stay consistent.
- Define one memorable differentiator and execute it deliberately.
- Match complexity to the design direction (maximalist vs minimalist).

## Execution principles
- Implement real, working UI code with production quality.
- Avoid generic patterns, default-looking palettes, and cookie-cutter compositions.
- Use cohesive typography, expressive color systems, and meaningful motion.
- Treat spacing, hierarchy, and visual detail as first-class concerns.

## Aesthetic guidance
- Typography: pair a distinctive display face with a readable body face.
- Color: use a strong palette with deliberate accents and CSS variables.
- Motion: prioritize a few high-impact animations over noisy micro-effects.
- Composition: use intentional asymmetry, overlap, and depth where appropriate.
- Detail: include textures, borders, gradients, or shadow systems that support the concept.

## Anti-patterns to avoid
- Overused, default AI aesthetics and predictable component layouts.
- Bland font choices and timid, evenly distributed color palettes.
- Visual decisions that do not reflect context, tone, or product goals.

Deliver interfaces that feel intentionally designed, memorable, and polished.
