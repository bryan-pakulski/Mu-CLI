Quick Start

1) List models

make models

2) Start a provider

make run-openai
# or
make run-gemini
# or local
make run-echo
# GUI
make run-web

3) Attach workspace

/workspace attach /path/to/repo
/workspace status

Core Concepts

Human-in-the-loop by design

Mutating tools are approval-gated and transparent. The assistant is optimized for collaboration, not hidden automation.

Agentic planning prompt

Planning guidance is auto-injected by default to make intent and tool choices explicit.

Selective context

The app indexes workspace files, but tools pull only targeted files into context on demand.

CLI Commands

CommandPurpose
/help, /tools, /tool-help <name>Discover available tools and args.
/workspace attach <path>, /workspace statusAttach/index repo and inspect status.
/models [provider], /model select <name>Inspect/switch model for current provider.
/approvals status, /approvals set ask|auto|denyControl mutating tool approval behavior.
/agentic statusCheck planning prompt mode status.
/debug status|on|offToggle debug traces for model tool-requests and tool executions.
/session status|list|new <name>|load <name>|delete <name>Manage persistent sessions from inside the app.

Flask GUI

Run make run-web and open http://localhost:5000.

The GUI provides a toggleable sidebar for provider/model/settings, inline session management (status/list/new/load/delete), workspace path, approval mode, and debug toggles.

Chat, traces, and per-turn cost report are available in one view for developer-in-the-loop workflows.

Workspace Context Flow

Recommended prompt pattern:

1) list_workspace_files(query="auth")
2) get_workspace_file_context(path="src/auth/service.py")
3) propose change plan
4) (optional) apply write/patch tools with approval

The index respects .gitignore and applies simple secret filtering for safer defaults.

Tooling & Approval Model

Read-only tools safe

  • read_file
  • list_workspace_files
  • get_workspace_file_context

Mutating tools approval-gated

  • write_file
  • apply_patch
  • git (constrained operations)

Providers & Models

Supported providers: echo, openai, gemini.

Use runtime flags:

--provider openai --model gpt-4o-mini
--provider gemini --model gemini-2.0-flash

Tool-calling is native for OpenAI/Gemini where supported.

Sessions, Pricing, and State

Debug mode: start with --debug or toggle with /debug on and /debug off.

When enabled, you see model tool requests (name/args/id) and tool execution traces/output previews.

Session persistence

Stored in .mu_cli/sessions/<session>.json. Resume is on by default. Disable with --no-resume.

Workspace memory

Workspace index + tool-run history are saved under .mu_cli/workspaces/.

Cost report

Each turn prints input/output/total token usage with estimated cost from .mu_cli/pricing.json.
